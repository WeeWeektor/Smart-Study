# Адаптивна логіка побудови навчального шляху

Знайти просто "схожі" курси за допомогою текстового аналізу (TF-IDF) недостатньо для побудови якісного освітнього
процесу. Платформа **Smart-Study** реалізує концепцію **адаптивного навчання** — система динамічно підлаштовує
складність рекомендованих матеріалів залежно від того, наскільки успішно студент засвоїв попередній матеріал.

Цей механізм реалізовано у методі `get_recommendations()` класу `CourseRecommender`.

## 1. Параметр складності (Level Mapping)

Для того щоб математично порівнювати складність курсів, текстові значення рівня підготовки з реляційної бази даних
трансформуються у числові ваги (`level_int`) під час роботи ML-пайплайну:

- `beginner` $\rightarrow 1$ (Базовий рівень)
- `intermediate` $\rightarrow 2$ (Середній рівень)
- `advanced` $\rightarrow 3$ (Просунутий рівень)

---

## 2. Алгоритм фільтрації за статусом проходження

Після обчислення косинусної схожості (`cosine_sim`) алгоритм отримує відсортований список найбільш релевантних за
тематикою курсів. Далі до цього списку застосовується адаптивний фільтр на основі статусу поточного курсу.

### 2.1. Сценарій успішного завершення (`status == 'passed'`)

**Педагогічна мета:** Просування вперед та поглиблення знань (Progression).

Якщо студент успішно склав усі тестування, система вважає, що він засвоїв поточний рівень матеріалу.

- **Логічне правило:** `candidate_level >= current_level`
- **Дія алгоритму:** Відбираються лише ті схожі за темою курси, складність яких є **такою ж або вищою**.
- **Приклад:** Якщо студент успішно завершив "Основи Python" (Level 1), йому будуть рекомендовані "Алгоритми на
  Python" (Level 2) або інші суміжні базові курси. Система відсікає надто прості матеріали, щоб студент не втрачав
  мотивацію.

### 2.2. Сценарій невдалого проходження (`status == 'failed'`)

**Педагогічна мета:** Закріплення матеріалу та заповнення прогалин у знаннях (Remediation).

Якщо студент не зміг набрати прохідний бал, рекомендувати йому складніші матеріали з цієї ж теми недоцільно — це
призведе до фрустрації.

- **Логічне правило:** `candidate_level <= current_level`
- **Дія алгоритму:** Відбираються схожі за темою курси, складність яких є **такою ж або нижчою**.
- **Приклад:** Якщо студент провалив "Просунутий Machine Learning" (Level 3), система порекомендує йому повторити "
  Основи математики для ML" (Level 2 або 1). Це дозволяє студенту зробити "крок назад", розібратися з фундаментальними
  поняттями і спробувати знову.

---

## 3. Захист від порожньої видачі (Fallback Strategy)

Оскільки жорстка фільтрація за рівнем (`>=` або `<=`) може призвести до того, що система відкине всіх знайдених
кандидатів (наприклад, якщо у базі є лише складні курси з певної вузької теми), передбачено механізм запобіжника (
Fallback).

1. Система намагається зібрати задану кількість рекомендацій (за замовчуванням `limit = 6`), проходячись по
   відсортованому масиву схожих курсів та застосовуючи правило `passed/failed`.
2. Якщо після застосування фільтрів список `recommended_indices` залишається порожнім, алгоритм **ігнорує рівень
   складності** і повертає просто найбільш схожі курси:
   `[df.iloc[score[0]]['id'] for score in sim_scores[:limit]]`
3. Якщо поточний курс взагалі відсутній у матриці (наприклад, щойно створений), система делегує задачу
   `CourseDBService`, який повертає глобально найпопулярніші курси (Cold Start).

## 4. Висновки

Такий підхід дозволяє перетворити звичайну систему пошуку схожого контенту на повноцінного "віртуального ментора".
Алгоритм не просто пропонує відео на ту ж тему, а враховує успішність студента, будуючи безпечну та ефективну траєкторію
навчання.
