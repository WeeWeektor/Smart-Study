# Роль Nginx: API Gateway, Reverse Proxy та SSL

У платформі **Smart-Study** Nginx виконує роль єдиної точки входу (API Gateway) для всіх зовнішніх запитів. Він
відповідає за маршрутизацію, безпеку (SSL/TLS), балансування та ефективну віддачу статичного контенту, знімаючи це
навантаження з Python-бекенду.

## 1. Архітектура безпеки та SSL/TLS (Термінація SSL)

Усі підключення до платформи є зашифрованими, а розшифрування трафіку (SSL Termination) відбувається на рівні Nginx.

- **HTTP to HTTPS Redirect:** Nginx слухає 80-й порт і примусово перенаправляє всі незахищені запити (
  `301 Moved Permanently`) на безпечний порт 443.
- **Сертифікати Let's Encrypt:** Використовуються згенеровані сертифікати (`fullchain.pem` та `privkey.pem`), які
  монтуються в контейнер.
- **Upload Limits:** Для забезпечення можливості завантаження важких матеріалів (наприклад, відеолекцій) встановлено
  `client_max_body_size 500M`.

## 2. Маршрутизація бекенду (Reverse Proxy)

Запити до бізнес-логіки розподіляються на upstream `asgi_backend` (контейнер Django, що працює на базі
Uvicorn/Gunicorn).

- **`/api/` (API Endpoints):** Основний канал зв'язку з бекендом. Nginx налаштовано на підтримку протоколу HTTP/1.1 та
  передачу заголовків `Upgrade` і `Connection "upgrade"`. Це критично важливо для коректної роботи ASGI-сервера та
  забезпечення можливості використання WebSockets у майбутньому.
- **`/admin/` (Панель адміністратора):** Винесено в окремий блок location для посилення контролю. Nginx примусово
  відключає кешування (`add_header Cache-Control "no-store, no-cache..."`) та передає реальні IP-адреси (`X-Real-IP`,
  `X-Forwarded-For`) для ведення точних логів безпеки в Django. Також реалізовано автоматичний редирект для очищення
  параметрів мови (`lang=`).

## 3. Обслуговування Frontend (Single Page Application)

Клієнтський додаток (React/Vite) компілюється в статичні файли (HTML/JS/CSS), які Nginx роздає з кореневої директорії
`/`.

- **SPA Роутинг:** Завдяки директиві `try_files $uri $uri/ /index.html`, Nginx не повертає помилку `404 Not Found`, якщо
  користувач перезавантажує сторінку на вкладеному URL (наприклад, `/courses/123`). Замість цього він віддає базовий
  `index.html`, дозволяючи React Router самостійно перехопити URL і відмалювати потрібний компонент сторінки.
- **Ізоляція адмінки:** Впроваджено регулярне вираз-правило `if ($request_uri ~* "^/admin/") { return 404; }` на рівні
  SPA, щоб уникнути конфліктів між фронтенд-роутингом та Django-адмінкою.
