# Інструкція з ручного оновлення SSL-сертифікатів (Runbook)

Для захисту передачі даних на платформі **Smart-Study** використовуються сертифікати Let's Encrypt. Оскільки термін їх
дії становить 90 днів, необхідно періодично виконувати їхнє ручне оновлення. Цей процес гарантує безпеку (HTTPS) та
стабільну роботу Reverse Proxy (Nginx).

## Алгоритм оновлення сертифіката

Коли наближається дата закінчення терміну дії сертифіката (рекомендується виконувати за 10-15 днів до завершення),
необхідно підключитися до сервера Azure через SSH та виконати наступні кроки.

### Крок 1. Зупинка Nginx (Звільнення порту 80)

Для того щоб Certbot міг підтвердити володіння доменом за допомогою тимчасового вебсервера (метод `--standalone`), порт
80 має бути вільним.

```bash
# Перехід у директорію проєкту
cd ~/smartStudy_project/smartStudy

# Зупинка контейнера nginx
docker-compose stop nginx
```

### Крок 2. Оновлення сертифіката

Виконання команди для автоматичного оновлення всіх сертифікатів, термін дії яких наближається до кінця. Ця команда
автоматично знайде всі налаштовані домени (наприклад, `smart-study.me` та `www.smart-study.me`).

```bash
sudo certbot renew
```

_Примітка:_ Якщо процес оновлення запитає спосіб підтвердження, оберіть опцію
`1: Spin up a temporary webserver (standalone)`.

### Крок 3. Запуск Nginx

Після успішного генерування нових ключів (`.pem` файли оновляться в директорії `/etc/letsencrypt/live/`), необхідно
відновити роботу вебсервера.

```bash
docker-compose start nginx
```

## Важливі примітки та вирішення проблем (Troubleshooting)

### 1. Верифікація оновлення

Щоб переконатися, що оновлення пройшло успішно, відкрийте платформу в браузері, натисніть на іконку "замочка" біля
URL $\rightarrow$ "Certificate is valid" та перевірте поле "Expires on". Дата повинна бути подовжена на 90 днів.

### 2. Примусове оновлення (Force Renewal)

Якщо сертифікат ще дійсний, але вимагається його негайне оновлення (наприклад, після зміни конфігурації або
компрометації), використовуйте прапорець примусового оновлення:

```bash
sudo certbot renew --force-renewal
```

### 3. Дії у випадку прострочення (Expired Certificate)

Якщо термін дії сертифіката минув, браузери почнуть блокувати доступ до сайту з попередженням "Your connection is not
private" / `ERR_CERT_DATE_INVALID`. Для відновлення роботи достатньо виконати стандартний алгоритм (Кроки 1-3).
